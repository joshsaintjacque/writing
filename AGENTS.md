# Repository Instructions

- This is a dependency-free static web app. Keep changes in plain HTML, CSS, and vanilla JavaScript unless the user explicitly asks to add a build tool or framework.
- For visible UI or interaction changes, run the app in a browser before handoff, capture a screenshot of the result, and inspect it visually to confirm the change renders as intended.
- When the app is running locally for validation, leave the server running and provide the user with a direct link (e.g. `http://localhost:8765/`) at the end of the turn.
- Commit each completed change as an atomic commit using Conventional Commits format, for example `feat(editor): add inline markdown rendering`.
- After committing completed changes, push the branch to GitHub so the GitHub Pages site receives the update. If the push fails, report the blocker clearly.
